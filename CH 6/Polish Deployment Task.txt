Polish + Deployment — Task List
Part 1: Polish (before touching deployment)

 Claude code review — have Claude review your current codebase for obvious issues (dead code, inconsistent patterns, missing error handling)
 UX polish pass — pick 2-3 concrete UX improvements and implement them (e.g. loading states, form validation feedback, empty states, mobile responsiveness)
 Write/update README — should cover: what the project does, tech stack, setup instructions, environment variables needed, how to run locally, folder structure overview
 Set up basic analytics — decide on a tool (Plausible, Umami, or even just server logs) and add tracking hooks for key user actions, ready to go live post-deploy
 Draft a demo script — 5-minute walkthrough: problem → solution → live demo → tech highlights → what's next. Practice it once out loud

Part 2: Deployment

 Choose a deployment platform — evaluate options with reasoning (e.g. Vercel/Netlify for frontend, Railway/Render/Fly.io for backend+DB, or a VPS if you want full control). Write down why you picked it (cost, DB support, ease of CI/CD, etc.)
 Deploy — get a live URL working end-to-end (frontend + backend + database all talking to each other in production)
 Security audit — before/after deploy:

 Confirm no secrets are hardcoded; everything sensitive is in .env and .env is gitignored
 Check CORS config is not wide open (*) in production
 Verify JWT secrets are strong and not the dev default
 Review any user input paths for injection risks (SQL, XSS)
 If you're using AI agents/MCP in the app itself, be aware of prompt injection risks


 Custom domain (optional) — if you have one, configure DNS (A/CNAME records) to point to your deployment; otherwise note the steps for later
 Share the live URL — post it somewhere (team channel, portfolio, etc.) as a milestone marker
